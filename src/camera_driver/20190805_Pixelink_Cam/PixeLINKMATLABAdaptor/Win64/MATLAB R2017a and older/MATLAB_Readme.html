PixeLINK's MATLAB Image Acquisition Toolkit Adaptor

Installation

The PixeLINK MATLAB adaptor has a few assumptions:

  • PixeLINK drivers and API have already been installed, v9 or higher is required.
  • MATLAB has already been installed
  • The MATLAB Image Acquisition Toolkit has already been installed

NOTE: The PixeLINK MATLAB R2017b adaptor has been tested with MATLAB R2017b.

NOTE: MATLAB R2015b is the last version of MATLAB that supports 32-bit operating systems. All later releases are 64-bit only.

To install the MATLAB adaptor, simply copy the following files in the PixeLINK MATLAB directory

 

For 32 Bit-OS:

  • C:\Program Files (x86)\PixeLINK\Matlab\Win32\ pixelinkimaq.dll and pixelinkimaq.imdf
  • To the MATLAB directory %ProgramFiles%\MATLAB\<Matlab Version>\toolbox\imaq\imaqadaptors.

 

For 64 Bit-OS:

  • C:\Program Files (x86)\ PixeLINK\Matlab\Win64\ pixelinkimaq.dll and pixelinkimaq.imdf
  • To the MATLAB directory %ProgramFiles%\MATLAB\<Matlab Version>\toolbox\imaq\imaqadaptors.

 

Example

For MATLAB 2017b 64-bit, copy the above-mentioned files from

C:\Program Files (x86)\PixeLINK\Matlab\Win64

to 

C:\Program Files\MATLAB\R2017b\toolbox\imaq\imaqadaptors

 

Registering the PixeLINK Adaptor in MATLAB

This only has to be done once. Once the dll is registered, MATLAB will know about it. 

 

For 32 Bit-OS:

>> imaqregister(['C:\Program Files\MATLAB\R2017b\toolbox\imaq\imaqadaptors\pixelinkimaq.dll'])
 

For 64 Bit-OS:

>> imaqregister(['C:\Program Files\MATLAB\R2017b\toolbox\imaq\imaqadaptors\pixelinkimaq.dll'])
 
ans = 
 
   cell
 
       'C:\Program Files\MATLAB\R2017b\toolbox\imaq\pixelinkimaq.dll'

Connecting to a Specific Camera With A Specific Format

>> vid=videoinput('pixelinkimaq', 1, 'MONO16_1280x1024')
 

You may also need to set the ReturnedColorSpace video object property depending on the camera model in use.

Getting a Snapshot

>>frame=getsnapshot(vid);

  

  
  
>>imtool(frame)
 

Matlab image commands do not automatically scale UINT16 images.  The solution is to scale the data before displaying it.

        

The following scales 10-Bit data in order to be viewed with the imagesc, image, imshow or imtool(recommended) commands: 

 
>> ScaledDoubleFrame=double(frame)/(2^10);
>> imtool(ScaledDoubleFrame)

>> vid=videoinput('pixelinkimaq')

   Acquisition Source(s):  input1 is available.
 
  Acquisition Parameters:  'input1' is the current selected source.
                           10 frames per trigger using the selected source.
                           'RGB24_1024x768' video data to be logged upon START.
                           Grabbing first of every 1 frame(s).
                           Log data to 'memory' on trigger.
 
      Trigger Parameters:  1 'immediate' trigger(s) on START.
 
                  Status:  Waiting for START.
                           0 frames acquired since starting.
                           0 frames available for GETDATA.
 
>> triggerConfigs = triggerinfo(vid)
 
triggerConfigs = 
 
1x4 struct array with fields:
    TriggerType
    TriggerCondition
    TriggerSource
 
>> triggerConfigs(3)
 
ans = 
 
         TriggerType: 'hardware'
    TriggerCondition: 'fallingEdge'
       TriggerSource: 'externalHardwareTrigger'
 
>> triggerconfig(vid, triggerConfigs(3))
>> start(vid)
>> isrunning(vid)
 
ans =
 
     1
 
>> islogging(vid)
 
ans =
 
     0
 
>> frame=getsnapshot(vid)     


The call to getsnapshot will block until a hardware trigger is received by the camera.
